# Копирование архива на сервер по шагам (логин root, SSH-ключ)

На сервере пользователь **root**. Значит, в команде везде пишем **root**, а не user.

---

## Шаг 1. Команда с правильным логином

В терминале на своём Mac вводишь **ровно эту** команду (подставь путь к архиву, если он в другом месте):

```text
scp /Users/deniscivkunov/wb-finance.zip root@194.87.96.144:/root/
```

Что здесь:
- **scp** — команда копирования по SSH
- **/Users/deniscivkunov/wb-finance.zip** — файл у тебя на Mac
- **root@194.87.96.144** — логин **root** на сервере с IP 194.87.96.144
- **/root/** — папка на сервере, куда положить файл (домашняя папка root)

Нажми Enter.

---

## Шаг 2. Если спросит про ключ или пароль

**Вариант А: ключ уже добавлен на сервер**

- Если ты уже добавлял свой публичный ключ в `/root/.ssh/authorized_keys` на новом сервере, то пароль спрашивать не должны — файл скопируется сам.
- Если всё равно просит пароль — вводи пароль от пользователя **root** на сервере (тот, что дал хостинг при создании сервера).

**Вариант Б: ключ на сервер ещё не добавлял**

1. На Mac посмотри, какой у тебя публичный ключ (обычно так):
   ```text
   cat ~/.ssh/id_rsa.pub
   ```
   или:
   ```text
   cat ~/.ssh/id_ed25519.pub
   ```
   Скопируй всю строку (начинается с `ssh-rsa` или `ssh-ed25519`).

2. Зайди на сервер **по паролю** (один раз):
   ```text
   ssh root@194.87.96.144
   ```
   Введи пароль root, когда попросит.

3. На сервере создай папку для ключей и добавь свой ключ:
   ```text
   mkdir -p ~/.ssh
   echo "СЮДА_ВСТАВЬ_СКОПИРОВАННУЮ_СТРОКУ_КЛЮЧА" >> ~/.ssh/authorized_keys
   chmod 700 ~/.ssh
   chmod 600 ~/.ssh/authorized_keys
   exit
   ```
   Вместо `СЮДА_ВСТАВЬ_...` — та самая длинная строка из `id_rsa.pub` или `id_ed25519.pub`.

4. Потом снова на Mac запусти копирование:
   ```text
   scp /Users/deniscivkunov/wb-finance.zip root@194.87.96.144:/root/
   ```
   Теперь пароль не должен спрашивать (будет использован твой ключ).

---

## Шаг 3. Проверка на сервере

Зайди на сервер:

```text
ssh root@194.87.96.144
```

Проверь, что файл на месте:

```text
ls -la /root/wb-finance.zip
```

Должен появиться файл и его размер. Всё ок — архив перенесён.

---

## Кратко

| Что было не так | Как правильно |
|-----------------|---------------|
| В команде было **user@** | Нужно **root@** (логин на твоём сервере — root). |
| Пароль не подходил | Либо ввести пароль root с хостинга, либо один раз добавить свой SSH-ключ в `authorized_keys` на сервере — тогда scp будет без пароля. |

Команда для копирования с твоим сервером:

```text
scp /Users/deniscivkunov/wb-finance.zip root@194.87.96.144:/root/
```
