# Что сделано и что делать дальше

## Уже сделано

1. **Docker Compose** — одной командой поднимаются PostgreSQL, Redis, API (FastAPI), Celery.
2. **Таблицы в БД** — добавлены модели и миграции для всех таблиц:
   - `users` — пользователи (email, пароль, API-ключ WB)
   - `articles` — артикулы и себестоимость
   - `raw_sales` — сырые продажи с WB
   - `raw_ads` — сырые данные по рекламе
   - `pnl_daily` — P&L по дням
   - `funnel_daily` — воронка по дням и артикулам

3. **Миграции** — при каждом запуске API миграции применяются сами (таблицы создаются/обновляются в PostgreSQL).

---

## Что делать тебе сейчас

### 1. Пересобрать и запустить контейнеры

После добавления миграций образ нужно пересобрать.

**В терминале (в папке проекта):**

Останови старый запуск, если он ещё работает: нажми **Ctrl + C** в окне, где был запущен `docker compose up`.

Потом введи:

```text
docker compose up --build
```

Подожди, пока всё поднимется. В логах должно появиться что-то вроде:

- `api-1 | INFO: Application startup complete.`
- Без ошибок про миграции (значит, таблицы созданы).

### 2. Проверить, что БД создана

В браузере открой:

```text
http://localhost:8000/health
```

Должно быть: `{"status":"ok"}`.

Если так и есть — API работает и миграции при старте отработали, таблицы в PostgreSQL созданы.
