# Бэклог: соответствие GAS и порядок работ

## Приоритет 1 — Лоадеры и «не первый вход» (как в GAS)

1. ~~**B2/C4 — Лоадер «7 дней» неблокирующий**~~ ✅  
   При не первом входе дашборды показываются сразу после проверки `has_data`. Плашка «Обновляем последние 7 дней…» — небольшая (баннер), не полноэкранный спиннер. Пользователь может смотреть данные, пока идёт фоновая догрузка.

2. ~~**C2 + C4 — Архив 2025**~~ ✅  
   - Backend: `POST /sync/backfill/2025` — догрузка 2025 по месяцам (sync_sales + sync_ads).  
   - Frontend: неблокирующая жёлтая плашка «Фоновая загрузка 2025…», опрос state раз в 15 сек до появления has_2025.

---

## Приоритет 2 — Устойчивость и UX

3. ~~**D3 — Фильтры дат без мигания**~~ ✅  
   Полноэкранный лоадер показывается только при первом запросе (когда данных ещё нет). При смене дат отображаются старые данные до прихода новых — без мигания.

4. ~~**E1 — Поведение при F5 (кэш при обновлении страницы)**~~ ✅  
   `CacheContext`: чтение кэша из `localStorage` при монтировании, запись при получении данных с API. При F5 табы показывают данные из кэша, затем фоновый запрос обновляет их. При выходе кэш очищается.

---

## Приоритет 3 — По желанию

5. ~~**E2 — Обновление себестоимости и пересчёт**~~ ✅  
   После сохранения себестоимости вызывается `triggerSyncRecalculate(dateFrom, dateTo)` и `onRefresh()` — дашборд и артикулы обновляются с учётом новых себестоимостей.
