# Как запустить проект (пошагово)

Инструкция для запуска всего одной командой. Ничего программировать не нужно — только повторять шаги.

---

## Шаг 0. Что должно быть установлено

На компьютере должен стоять **Docker Desktop**.

- Если ещё не установлен: скачай с https://www.docker.com/products/docker-desktop/
- Установи, запусти программу Docker Desktop и дождись, пока внизу слева будет зелёный индикатор «Docker is running». Без этого дальше не получится.

---

## Шаг 1. Открыть «Терминал» (чёрное окно для ввода команд)

- **Mac:** нажми `Cmd + Пробел`, введи `Terminal` и нажми Enter. Или: Программы → Утилиты → Terminal.
- **Windows:** нажми `Win + R`, введи `cmd` и нажми Enter. Или в поиске Windows введи «Командная строка» и открой её.

Откроется окно с текстом и мигающим курсором — туда ты будешь вводить команды.

---

## Шаг 2. Перейти в папку проекта

В терминале нужно «зайти» в папку, где лежит проект.

**Введи по очереди (после каждой строки нажимай Enter):**

```text
cd
```

Потом (подставь свой путь к папке, где у тебя лежит wb-finance):

```text
cd /Users/deniscivkunov/wb-finance
```

Если проект лежит в другом месте (например на Рабочем столе), то вместо этого пути напиши свой, например:

```text
cd C:\Users\ТвоёИмя\Desktop\wb-finance
```

Главное — путь должен вести именно в папку, внутри которой лежат файлы `docker-compose.yml` и папка `backend`.

Проверка: введи:

```text
dir
```

(На Mac можно: `ls`). Должны появиться названия файлов и папок, среди них — `docker-compose.yml` и `backend`. Если их видишь — ты в нужной папке.

---

## Шаг 3. Запустить всё одной командой

Одна команда поднимет базу, Redis, API и воркер.

**Введи ровно это (можно скопировать и вставить):**

```text
docker compose up --build
```

Нажми **Enter**.

- Первый раз будет долго: Docker скачает образы и соберёт проект (может быть 2–5 минут).
- В окне пойдёт текст — это логи. Ошибки обычно красным.

Дождись, пока не появится что-то вроде:

```text
api         | Uvicorn running on http://0.0.0.0:8000
```

и

```text
celery_worker | celery@... ready.
```

Когда видишь такое — всё запустилось.

**Окно не закрывай** — пока оно открыто, сервисы работают. Чтобы остановить — в этом же окне нажми **Ctrl + C** (на Mac тоже Ctrl + C).

---

## Шаг 4. Проверить, что API работает

Открой браузер (Chrome, Safari, любой) и в адресной строке введи:

```text
http://localhost:8000/health
```

Нажми Enter.

Должна открыться страница с текстом вроде: `{"status":"ok"}`. Значит, API запущен правильно.

---

## Шаг 5. Остановить проект

Когда захочешь всё выключить:

1. Вернись в окно терминала, где запускал `docker compose up --build`.
2. Нажми **Ctrl + C** (одновременно клавиши Ctrl и C).
3. Подожди несколько секунд, пока контейнеры остановятся.
4. Можно закрыть окно терминала.

Чтобы потом снова запустить — повтори **Шаг 1 → Шаг 3**.

---

## Если что-то пошло не так

**«docker: command not found» или «docker не является командой»**  
Значит, Docker не установлен или не запущен. Установи Docker Desktop и убедись, что он запущен (зелёный статус).

**«No such file or directory» при вводе `cd ...`**  
Ты не в той папке или путь написан с ошибкой. Проверь, что путь ведёт в папку с файлом `docker-compose.yml`. На Mac путь часто начинается с `/Users/...`, на Windows — с `C:\...`.

**Ошибки в логах про порт 5432 или 6379**  
Кто-то уже использует этот порт (например, другой PostgreSQL или Redis). Закрой другие программы, которые могут занимать порты, или напиши в чат — подскажем, как поменять порты в `docker-compose.yml`.

**Страница http://localhost:8000/health не открывается**  
Убедись, что в терминале видно «Uvicorn running» и ты не нажал Ctrl+C. Попробуй подождать ещё 30 секунд и обновить страницу (F5).

---

## Кратко: что вводить по порядку

1. Открыть **Терминал**.
2. Ввести: `cd /Users/deniscivkunov/wb-finance` (или свой путь к папке проекта).
3. Ввести: `docker compose up --build`.
4. Дождаться сообщений про Uvicorn и Celery.
5. В браузере открыть: `http://localhost:8000/health` — должно быть `{"status":"ok"}`.
6. Чтобы остановить: в терминале нажать **Ctrl + C**.

Всё. Дальнейшие шаги (миграции, авторизация и т.д.) будем разбирать отдельно, тоже по шагам.
