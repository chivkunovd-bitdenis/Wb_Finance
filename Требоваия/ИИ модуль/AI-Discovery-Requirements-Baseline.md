# AI module — Discovery answers → Requirements Baseline (2026-05-10)

Requirements Baseline:
- Problem / user intent (1–2 sentences):
  Нужен ИИ‑модуль, который ежедневно создаёт пользователю задачи/гипотезы по развитию карточек WB, опираясь на конкурентные метрики и данные оцифровки, и поддерживает обновление конкурентных отчётов через WB (без ручных Excel‑выгрузок).
- Primary user outcome (what user can do/see):
  Пользователь видит в UI (страница “ИИ”) списки задач/гипотез и конкурентные отчёты; отчёты обновляются по расписанию; задачи/гипотезы создаются автоматически (идемпотентно).
- Success criteria (observable; “it works”):
  - Есть UI‑entry “ИИ” как отдельная страница.
  - Конкурентные отчёты обновляются по расписанию без ручного Excel.
  - Ежедневная аналитика создаёт задачи/гипотезы по правилам ТЗ, без дублей.
  - Пользовательское “согласие” действует 3 дня (отдельно фиксируем).
- Scope (in):
  - UI entry point: отдельная страница “ИИ” (Q1=A).
  - Источники данных: конкурентные метрики и остальные данные будут получаться **отдельными методами WB** (Q2).
  - MVP4: обновление по расписанию (Q3=по расписанию), согласие на 3 дня (Q4=B), idempotency “raw immutable + aggregated upsert” (Q5=C).
  - Никаких реальных Excel‑выгрузок/импортов: manual import существует только как временная заглушка до готовности интеграции (важное уточнение).
  - Доступы: используем **WB API key** (Q6=A).
  - Definition of Done: “в MVP готовы все пункты” (Q7) — трактуем как: все пункты ТЗ по конкурентным отчётам/аналитике/акциям должны быть реализованы в MVP, без выноса в EXP.
- Out of scope:
  - Не зафиксировано отдельно; “автосоздание карточки” и “парсинг выдачи” остаются в EXP только если явно не включаются в “все пункты”.
- Constraints (tech/legal/security/time):
  - Только WB API methods + расписание; без Excel.
  - Нужна строгая идемпотентность и лимиты по затратным операциям сравнения.
- UX constraints (where it appears; navigation; pattern constraints):
  - Отдельная страница “ИИ”, не drawer и не карточка товара.
- Non-goals / what must NOT change:
  - Нельзя вводить Excel‑флоу как “реальную” функциональность (manual import — только stub).
  - Нельзя требовать логин/пароль WB кабинета (только API key).
- Open questions (blocking only; choice-based):
  - **Q7 уточнение**: “все пункты” включает ли EXP1/EXP2 (автосоздание карточки/парсинг выдачи) или речь только про пункты конкурентных отчётов + ежедневной аналитики + акции/маржа/автоисключение?
    - A) включает EXP1+EXP2
    - B) включает только конкурентов+аналитику+акции
    - Recommended: B (минимизировать риск и объём интеграций)
- Assumptions (explicit):
  - “Отдельные методы WB” доступны или будут реализованы как отдельные интеграции без Playwright.
  - Модель “согласия на 3 дня” будет храниться в БД и применяться при расписании обновления.
