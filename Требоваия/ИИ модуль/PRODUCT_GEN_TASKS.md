# Полная генерация товара (ИИ) — бэклог **v2**

**Цель порядка:** сначала доводим до конца **каскад генерации изображений** (ввод только референсы + пользовательский текст + зашитый шаблон промпта на стороне пайплайна). **Отдельной веткой позже** — заполнение карточки товара (размеры, цена, артикул, …) и публикация в WB.

**План с контрактами:** [`PRODUCT_GEN_PLAN.md`](./PRODUCT_GEN_PLAN.md) — §2–4 синхронизированы с v2 (PG-A.0).

**Товар / WB (отдельный бэклог):** [`PRODUCT_GEN_PRODUCT_STREAM.md`](./PRODUCT_GEN_PRODUCT_STREAM.md) — не входит в текущую итерацию IMAGE; не смешивать с задачами ниже.

Легенда: `[ ]` не начато · `[~]` в работе · `[x]` готово · **DoD** = критерий готовности.

**Правило для Builder (IMAGE):** идти **сверху вниз по фазам A → B → C** (и **E.2** при необходимости); **не** открывать поток товара из [`PRODUCT_GEN_PRODUCT_STREAM.md`](./PRODUCT_GEN_PRODUCT_STREAM.md), пока не закрыт минимум **B.1–B.3** (реальная генерация 4 кадров и наблюдаемый результат в сервисе/через прокси).

## Зачем v2 (что вырезали из старой логики)

| Было (старый PG-2.* нарратив) | Стало |
|------------------------------|--------|
| Один мастер: сначала все поля карточки, потом «Создать» и старт | **Старт каскада** не зависит от артикула, габаритов, размеров, цены — только референсы + текст пользователя (и шаблон в воркере) |
| «Создать черновик» = сразу весь контракт карточки | **Создание/дозаполнение товара** — отдельный шаг после фото (UI уже частично так; бэк по **A**; товар/WB — только [`PRODUCT_GEN_PRODUCT_STREAM.md`](./PRODUCT_GEN_PRODUCT_STREAM.md)) |
| Фаза 4 в старом файле шла после «полной формы» | Фаза **B** — реализация LLM/Image **без** требования полей карточки на входе run |
