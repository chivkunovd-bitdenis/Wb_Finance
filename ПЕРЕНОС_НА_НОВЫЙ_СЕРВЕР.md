# Перенос проекта на другой сервер

Всё, что уже есть (код, Docker, при необходимости — данные БД), можно перенести на новый сервер за несколько шагов.

---

## Что у тебя есть сейчас

- **Код проекта** — папка `wb-finance` (backend, docker-compose, миграции, старый Index.html/Code.js и т.д.).
- **Docker** — поднимается командой `docker compose up`.
- **Данные** (если уже что-то заливал в БД) — лежат в томе PostgreSQL внутри контейнера.

Цель: то же самое работало на новом сервере (новый IP/домен).

---

## Вариант А: Данных в БД ещё нет (только код)

Самый простой случай: пользователей и данных в PostgreSQL пока не было.

### На старом компьютере (или откуда вёл разработку)

1. **Упаковать весь проект в архив**
   - Зайди в папку **над** `wb-finance` (например, в `Users/deniscivkunov/`).
   - В терминале введи (подставь свой путь, если папка в другом месте):

   ```text
   cd /Users/deniscivkunov
   zip -r wb-finance.zip wb-finance
   ```

   Появится файл `wb-finance.zip` рядом с папкой `wb-finance`.

2. **Перенести архив на новый сервер**
   - Через **SFTP/SCP** (FileZilla, WinSCP, или команда `scp`).
   - Пример через терминал (подставь свой логин и новый адрес сервера):

   ```text
   scp /Users/deniscivkunov/wb-finance.zip user@НОВЫЙ_IP_АДРЕС:/home/user/
   ```

   Или залей архив через панель хостинга (файловый менеджер), если так удобнее.

### На новом сервере

3. **Подключись по SSH** к новому серверу (как обычно заходишь по SSH).

4. **Распаковать проект**

   ```text
   cd /home/user
   unzip wb-finance.zip
   ```

   (Если `unzip` нет: `sudo apt install unzip` на Ubuntu/Debian.)

5. **Установить Docker** (если ещё не стоит):
   - Ubuntu/Debian: инструкция с официального сайта https://docs.docker.com/engine/install/ubuntu/
   - Или одной командой (скопируй целиком):

   ```text
   curl -fsSL https://get.docker.com -o get-docker.sh && sudo sh get-docker.sh
   ```

6. **Запустить проект**

   ```text
   cd wb-finance
   docker compose up -d --build
   ```

7. **Проверить:** в браузере открой `http://НОВЫЙ_IP:8000/health` — должно быть `{"status":"ok"}`.

Готово: всё перенесено, код и Docker работают на новом сервере.

## Вариант Б: В БД уже есть пользователи/данные (нужно сохранить)

Если на старом сервере уже крутился проект и в PostgreSQL накопились данные — их нужно выгрузить и загрузить на новом.

### На старом сервере (где сейчас всё работает)

1. **Сделать дамп базы**

   ```text
   cd /путь/к/wb-finance
   docker compose exec postgres pg_dump -U wb_finance wb_finance > backup.sql
   ```

   В папке проекта появится файл `backup.sql` — это полная копия БД.

2. **Упаковать проект вместе с дампом**

   ```text
   cd /путь/к/папке_выше_wb-finance
   zip -r wb-finance-with-db.zip wb-finance
   ```

   В архив попадёт и `wb-finance/backup.sql`.

3. **Перенести архив на новый сервер** (как в варианте А — scp, SFTP или панель хостинга).

### На новом сервере

4. **Распаковать**, установить Docker (если нужно), как в шагах 3–5 варианта А.

5. **Сначала поднять контейнеры без восстановления БД**

   ```text
   cd wb-finance
   docker compose up -d --build
   ```

   Подожди 10–15 секунд, пока PostgreSQL запустится.

6. **Восстановить БД из дампа**

   ```text
   docker compose exec -T postgres psql -U wb_finance wb_finance < backup.sql
   ```

   Если команда ругается на существующие объекты — можно один раз пересоздать базу (удалит все данные в контейнере и создаст заново):

   ```text
   docker compose down
   docker volume rm wb-finance_postgres_data
   docker compose up -d
   ```
   Подожди 10 секунд, затем снова выполни команду с `backup.sql`.

7. **Проверить:** `http://НОВЫЙ_IP:8000/health` и проверь в приложении, что пользователи/данные на месте.

---

## Что поменять после переноса

- **IP в постановке/документах** — везде, где был старый российский IP (например 193.124.114.77), замени на новый адрес сервера.
- **Домен** (если будет) — потом настроишь в панели хостинга и при необходимости в `docker-compose` (порты уже открыты 8000, 5432, 6379 при необходимости).
- **Переменные окружения** — если на новом сервере будут другие пароли БД или URL, их можно задать в `docker-compose.yml` в секции `environment` или через файл `.env` рядом с `docker-compose.yml`.

---

## Кратко

| Ситуация | Действия |
|----------|----------|
| **Только код, БД пустая** | Архив папки проекта → перенос на новый сервер → распаковать → `docker compose up -d --build` → проверить /health. |
| **Есть данные в БД** | На старом: `pg_dump` → сохранить `backup.sql` в папку проекта → архив → перенос. На новом: распаковать → `docker compose up -d` → `psql ... < backup.sql` → проверить. |

После переноса продолжаешь разработку уже на новом сервере (или снова с локального компьютера, но деплой — на новый адрес). Нейросети и внешние API будут доступны с этого сервера без ограничений российского хостинга.
